# Server Telemetry with Grafana

## Example: Grafana and Open Telemetry Collector

This guide integrates the Apollo MCP Server with the `grafana/otel-lgtm` docker container as an all-in-one local solution for testing Telemetry.

## Getting Started / Usage Guide

To start receiving telemetry, you need an OpenTelemetry Collector to which the MCP server can send data.

### 1. Run the Grafana stack using Docker

Start up the [Grafana LGTM](https://grafana.com/docs/opentelemetry/docker-lgtm/) stack with `docker`

```sh
docker run -p 3000:3000 -p 4317:4317 -p 4318:4318 --rm -ti grafana/otel-lgtm
```

This starts up the OTel Collector for data ingestion on port 4317 for gRPC and 4318 for HTTP.
The Grafana web application is started on port 3000 http://localhost:3000/ where you can log in using the default credentials `admin`/`admin`

### 2. Configure and Run the MCP Server

Start the MCP server and point it to the collector's OTLP endpoint using the following configuration.

```yaml
telemetry:
  exporters:
    metrics:
      otlp:
        endpoint: "http://localhost:4318/v1/metrics"
        protocol: "http/protobuf"
    tracing:
      otlp:
        endpoint: "http://localhost:4318/v1/traces"
        protocol: "http/protobuf"
```

Run your apollo-mcp-server binary or Docker container

```sh
./apollo-mcp-server --config ./config.yaml
```


### 3. Observe Telemetry

As you interact with the MCP server, you will see trace and metric data appear in the Grafana UI under Drilldown > Traces and Drilldown > Metrics
